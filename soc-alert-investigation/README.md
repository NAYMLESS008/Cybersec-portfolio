# SOC Alert Investigation

## Overview

This section contains SOC-style investigation notes and reports. The purpose is to demonstrate alert triage, evidence collection, log correlation, MITRE ATT&CK mapping, verdict classification, and response recommendations.

## Investigation Workflow

1. Review the alert name, severity, timestamp, and affected asset.
2. Identify source and destination details.
3. Review related logs around the event time.
4. Check for repeated or abnormal behaviour.
5. Classify the activity as true positive, false positive, or requiring further investigation.
6. Map observed behaviour to MITRE ATT&CK where appropriate.
7. Recommend containment, eradication, recovery, or monitoring actions.
